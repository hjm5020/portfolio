인사 자원 시스템 — 멀티 테넌시 아키텍처 설계

2024.07 ~ 2024.12 · 주식회사 디케이테크인

배경

다수의 조직(테넌트)이 하나의 인사 자원 관리 시스템을 공유해야 하는 요구사항이 있었습니다. 각 조직마다 별도 시스템을 구축하면 유지보수 비용이 기하급수적으로 증가하고, 조직 추가 시마다 신규 배포가 필요한 구조적 한계가 있었습니다.

문제 정의

  • 조직마다 독립적인 데이터 격리가 필요하지만, 인프라 비용은 최소화해야 함
  • 새로운 조직 추가 시 코드 변경 없이 확장 가능한 구조가 필요
  • 기존 동기식 처리 방식으로 인해 대량 데이터 처리 시 사용자 응답 지연 발생

해결 과정

1. 테넌트 분리 전략 선택

멀티 테넌시 구현 방식은 크게 세 가지가 있었습니다. DB 인스턴스 분리, 스키마 분리, 그리고 공유 테이블 + 테넌트 컬럼 방식. 각 방식의 트레이드오프를 분석한 결과, 프로젝트 규모와 비용 효율성을 고려하여 적합한 분리 전략을 선택했습니다.

테넌트 분리 방식 비교

DB 분리
격리성 ↑ 비용 ↑ vs 스키마 분리
균형 vs 컬럼 분리
비용 ↓ 격리성 ↓

2. 비동기 처리 전환

대량 데이터 처리(의료비 일괄 정산, 조직 정보 동기화 등)에서 동기식 처리가 사용자 응답 시간을 크게 지연시키는 문제가 있었습니다. 이를 비동기 백그라운드 처리로 전환하여 사용자 요청에 대한 즉시 응답을 보장하고, 실제 처리는 백그라운드에서 수행하도록 개선했습니다.

처리 방식 변경

사용자 요청 → 즉시 응답 반환 → 백그라운드 처리 → 완료 알림

결과

  • 다양한 조직 규모에 대응 가능한 유연한 멀티 테넌시 시스템 구축
  • 신규 조직 추가 시 설정만으로 온보딩 가능한 확장성 확보
  • 비동기 전환을 통해 대량 처리 시에도 사용자 응답성 유지

회고

이 프로젝트를 통해 시스템 아키텍처 설계 단계에서의 의사결정이 이후 개발 과정과 유지보수에 얼마나 큰 영향을 미치는지 체감할 수 있었습니다. 단순히 기능을 구현하는 것을 넘어, 확장성과 비용 효율성 사이에서 균형을 찾는 경험을 쌓을 수 있었습니다.
