카카오 휴양지 예약 시스템 — Redis 분산 락 기반 동시성 제어

2024.01 ~ 2024.05 · 주식회사 디케이테크인

배경

카카오 임직원 대상 휴양지 예약 시스템에서, 인기 휴양지의 예약 오픈 시 동시에 다수의 사용자가 예약 요청을 보내는 상황이 빈번하게 발생했습니다. 기존 로직에서는 동일 사용자가 짧은 시간 내 중복 요청을 보낼 경우 이를 제대로 차단하지 못하는 문제가 있었습니다.

문제 정의

  • 동시 요청 시 동일 사용자의 중복 예약이 발생할 수 있는 구조적 취약점
  • DB 레벨의 유니크 제약만으로는 사용자 경험이 좋지 않음 (에러 메시지 노출)
  • 단순 synchronized 처리는 멀티 서버 환경에서 동작하지 않음

문제 상황

사용자 A 요청 1 → 서버 1 → 중복 예약 발생
사용자 A 요청 2 → 서버 2 → 중복 예약 발생

해결 과정

1. 분산 락 도입 검토

멀티 서버 환경에서의 동시성 제어를 위해 분산 락 방식을 검토했습니다. DB 기반 비관적 락, Redis 기반 분산 락, ZooKeeper 기반 락 등의 선택지가 있었고, 이미 세션 관리에 Redis를 사용하고 있어 추가 인프라 없이 적용 가능한 Redis 분산 락을 선택했습니다.

2. 사용자 Key 기반 락 설계

예약 요청 시 사용자 ID를 기반으로 Redis에 락 키를 생성하여, 동일 사용자의 동시 요청을 서버와 관계없이 차단하도록 설계했습니다. TTL을 설정하여 장애 상황에서도 락이 자동으로 해제되도록 안전장치를 마련했습니다.

해결 후 흐름

예약 요청 → Redis 락 획득 → 예약 처리 → 락 해제

3. 세션 및 캐싱 전략 통합

분산 락 도입과 함께 기존에 분산되어 있던 Redis 활용 방식을 통합 정리했습니다. 사용자 로그인 세션 관리, 자주 조회되는 예약 현황 데이터 캐싱, 그리고 분산 락까지 Redis 하나로 일관성 있게 관리하도록 개선했습니다.

결과

  • 멀티 서버 환경에서 동일 사용자의 중복 예약 문제 해결
  • TTL 기반 자동 해제로 장애 상황에서의 데드락 방지
  • Redis 활용 전략 통합으로 인프라 관리 포인트 단순화

회고

단일 서버에서는 간단한 동기화로 해결될 문제도, 분산 환경에서는 완전히 다른 접근이 필요하다는 것을 실감한 프로젝트였습니다. Redis 분산 락이라는 해결책뿐만 아니라, 기존에 흩어져 있던 Redis 활용 방식을 함께 정리할 수 있었던 의미 있는 경험이었습니다.
