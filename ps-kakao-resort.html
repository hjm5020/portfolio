카카오 휴양지 예약 시스템 — Redis 분산 락 기반 동시성 제어

2024.01 ~ 2024.05 · 주식회사 디케이테크인

배경

카카오 임직원 대상 휴양지 예약 시스템에서, 인기 휴양지 예약 오픈 시 동일 사용자의 중복 예약이 발생하는 실제 장애가 있었습니다. 임시 조치로 장애를 처리한 뒤, 근본적인 원인을 분석하여 멀티 서버 환경에서도 안전하게 동작하는 동시성 제어 방안을 찾아 적용했습니다.

문제 정의

  • 동일 사용자가 짧은 시간 내 중복 요청을 보낼 경우, 애플리케이션 레벨의 중복 체크만으로는 동시 요청을 차단하지 못함
  • 단순 synchronized 처리는 멀티 서버 환경에서 서버 간 상태를 공유하지 않아 동작하지 않음

장애 상황

사용자 A 요청 1 → 서버 1 → 중복 예약 발생
사용자 A 요청 2 → 서버 2 → 중복 예약 발생

해결 과정

1. 분산 락 도입

멀티 서버 환경에서의 동시성 제어 방안을 조사하는 과정에서 Redis 기반 분산 락 개념을 알게 되었고, 이미 세션 관리에 Redis를 사용하고 있어 추가 인프라 없이 적용 가능한 Redisson 기반 분산 락을 도입했습니다.

2. 락 키 설계 및 구현

락 키를 userId:startDate:endDate 패턴으로 설계하여, 동일 사용자가 같은 날짜에 중복 예약을 시도하는 것을 서버와 관계없이 차단하도록 했습니다. tryLock(10, TimeUnit.MILLISECONDS)으로 10ms 내에 락을 획득하지 못하면 즉시 "같은 날짜에 두번 이상 신청하실 수 없습니다"로 응답하여 사용자에게 빠른 피드백을 제공합니다.

락 획득 후에는 패널티 체크, 연간 신청일수 제한, 잔여 객실 확인 등 비즈니스 검증을 순차적으로 수행하고, finally 블록에서 isHeldByCurrentThread() 확인 후 안전하게 락을 해제하여 장애 상황에서의 데드락을 방지했습니다.

해결 후 흐름

예약 요청 → Redisson 락 획득 → 비즈니스 검증 + 예약 처리 → finally 락 해제

결과

  • 멀티 서버 환경에서 동일 사용자의 중복 예약 장애 재발 방지
  • 10ms 타임아웃으로 중복 요청 시 빠른 사용자 피드백 제공
  • finally 블록 기반 안전한 락 해제로 데드락 방지

회고

실제 장애를 경험하면서, 단일 서버에서는 문제가 없던 로직이 멀티 서버 환경에서는 완전히 다른 접근이 필요하다는 것을 체감했습니다. 임시 조치로 급한 불을 끈 뒤 근본 원인을 분석하고, Redisson 분산 락으로 구조적으로 해결하는 과정을 통해 장애 대응과 후속 개선의 중요성을 배울 수 있었습니다.
